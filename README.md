## How to Use

### Method 1: Claude.ai Projects (Recommended)
1. Go to [claude.ai](https://claude.ai) → **Projects**
2. Create or open a project
3. Click **Add Content** → upload `SKILL.md`
4. Optionally upload relevant `references/*.md` files for deeper domain coverage
5. Start chatting — Claude now has full IoT protocol expertise

### Method 2: OpenClaw Skills
1. Download `iot-protocols-expert.skill` from [Releases](https://github.com/svdwalt007/Best-IoT-Protocols-Skills/releases)
2. Install via OpenClaw skill manager or drop in your skills directory
3. The skill activates automatically on IoT protocol queries

### Method 3: API System Prompt
```python
import anthropic

with open("SKILL.md") as f:
    skill = f.read()

client = anthropic.Anthropic()
response = client.messages.create(
    model="claude-opus-4-6",
    system=skill,
    messages=[{"role": "user", "content": "Compare NB-IoT NIDD vs standard bearer for water meter deployments"}]
)
```

### Method 4: Enhanced Context (Individual Reference Files)
For deep domain work, add specific reference files alongside SKILL.md:

## Critical Notes Embedded in This Skill

### DTLS CID (RFC 9146)
Extension type **54** is correct (ratified March 2022). Extension type **53** is the obsolete draft. Mixing causes **silent interop failures**. `tls12_cid` ContentType = **25**. RFC 9853 adds Return Routability Check (DTLS RRC) for DTLS 1.3. See `references/security-provisioning.md`.

### SGP.32 eSIM Objects
LwM2M Objects: **504** (RSP — 21 resources for eSIM lifecycle) and **3443** (eSIM IoT — ASN.1 pipe between eIM and IPA). **Object 146 is NOT an eSIM object** — it's a page number in the SGP.32 spec. **IPAd** = in device OS; **IPAe** = inside eUICC.

### Wireless M-Bus / OMS
EN 13757-4. T/S/R/C transmission modes at 868 MHz; N mode at 169 MHz (Wize). Pairs with DLMS/COSEM application layer. 100M+ water/gas/heat meters deployed across EU.

### EU CRA + RED Article 3.3
Mandatory for all wireless IoT devices sold in EU from 2025/2026. Not optional. See `references/security-provisioning.md`.
